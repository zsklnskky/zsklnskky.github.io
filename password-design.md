# 🔐 PASSWORD — Дизайн-документ

> Менеджер паролей в стиле 1Password / Bitwarden, адаптированный под наш стек (single-file HTML + Firebase).

---

## 🎯 Что мы строим (одной фразой)

Безопасный менеджер паролей, где **всё шифруется на клиенте** мастер-паролем. Firebase видит только зашифрованный блоб — даже если кто-то получит доступ к твоей Firestore-базе, без мастер-пароля он не прочитает ничего.

Это та же модель, что у 1Password и Bitwarden — называется **"zero-knowledge"**: сервер ничего не знает.

---

## 🔒 Архитектура безопасности (ключевое)

### Two-key модель

У тебя **2 пароля**:

1. **Google-логин** — авторизует доступ к твоему Firestore-документу
2. **Мастер-пароль** — расшифровывает содержимое

**Зачем оба?** Google защищает от чужих, мастер-пароль защищает даже от тебя в случае, если Google-аккаунт взломают. Если хакер залез в твой Google, он увидит Firestore с зашифрованным блобом — но без мастер-пароля прочитать не сможет.

### Что хранится в Firestore

```js
wallets/<uid>     ← данные WALLET (не зашифрованы, обычные данные)
users/<uid>       ← данные FOCUS (не зашифрованы)
passwords/<uid>   ← {                                  // ВСЁ ЗАШИФРОВАНО
  salt: "base64...",            // случайные 16 байт, не секрет
  iterations: 200000,           // PBKDF2 итерации
  verification: "encrypted...", // проверка правильности master pw
  vault: "encrypted blob..."    // массив записей, шифрован AES-GCM-256
  updatedAt: timestamp
}
```

### Криптография (Web Crypto API, встроена в браузер)

- **AES-GCM-256** для шифрования записей
- **PBKDF2 SHA-256, 200K итераций** для деривации ключа из мастер-пароля
- **Random IV (96 bit)** для каждого шифрования
- Salt уникальный для юзера, генерится один раз при создании vault'а

### Если забыл мастер-пароль

**Данные потеряны. Восстановить нельзя.** Это цена настоящей безопасности — ни мы, ни Google не знают твой мастер-пароль.

**Решение:** записать мастер-пароль на бумажке + положить в надёжное место (сейф, банковская ячейка). 1Password предлагает "emergency kit" — печатный документ с инструкциями восстановления.

| Тип | Поля |
|---|---|
| 🔑 **Login** | title, username, password, url, totp, notes |
| 💳 **Card** | title, cardholder, number, expiry, cvc, pin, notes |
| 📝 **Note** | title, content (markdown), tags |
| 👤 **Identity** | имя, паспорт, ИНН, адрес, телефон, email |
| 🔐 **API Key** | title, service, key, secret, expires |
| 🌐 **Server** | title, host, port, username, password, ssh-key |

- ❌ Native iOS/Android — будем веб-only

## ⚠️ Важные UX-моменты

### 1. Clipboard auto-clear
Когда копируешь пароль — таймер 30 сек, потом буфер чистится. Иначе следующее приложение, в которое вставишь, увидит твой пароль.

### 2. Auto-lock
После 15 минут неактивности vault блокируется. Нужно снова ввести мастер-пароль.

### 3. Скрытие при переключении вкладок
Когда вкладка теряет фокус — содержимое размывается (`filter: blur`). Защита от shoulder-surfing.

### 4. Никакого "remember me"
Мастер-пароль не сохраняется. Каждый раз ввод. (В v3 можно добавить WebAuthn).

### 5. Master password не отображается ни в каких логах
Web Crypto API работает с CryptoKey-объектами, raw-ключ не доступен после деривации.

### 6. Версионирование схемы
В vault'е поле `version: 1`. Если в будущем поменяется формат — `migrate()` обновит при разблокировке.

## 🔐 Threat model (от кого защищаемся)

| Атакующий | Защита |
|---|---|
| **Сосед видит экран** | Auto-blur при потере фокуса + пароли по умолчанию скрыты |
| **Кто-то крадёт ноут** | Master password + auto-lock |
| **Взлом Google-аккаунта** | Master password шифрует всё |
| **Утечка из Firebase** | Только зашифрованный блоб |
| **Кейлоггер на ноуте** | Не защищаемся, это вне нашей модели |
| **Государство с физ. доступом** | Не защищаемся, нужен Tails OS + offline manager |
